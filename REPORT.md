|---|---|---|
| **전자문서 및 전자거래 기본법** | 제4조(전자문서의 효력), 제5조(작성·송신·수신), 제7조(보관) | 높음 |

> **참고**: 본 시스템은 상거래 전자계약이 아닌 **공교육 행정 동의서 수집** 시스템입니다. 「전자문서법」의 전자문서 효력 인정 요건을 기준으로 부합도를 평가하되, 교육 현장의 현실적 수준을 감안합니다.

### 1.2 전자문서 요건 부합도

| 요건 | 법적 기준 | 현재 구현 상태 | 부합도 | 비고 |
|---|---|---|:---:|---|
| **작성자 확인** | 전자문서법 제5조 | 보호자 성명 직접 입력 + "동의합니다" 확인 문구 타이핑 | **○** | 공인인증서 수준은 아니나, 교육 현장 동의서의 일반적 수준 충족 |
| **전자서명** | 전자서명법 제2조 | 성명 입력 + 확인 문구 + 선택적 자필 서명(캔버스) | **△** | 공인전자서명(공동인증서)은 미적용. 간편서명(타이핑+서명패드) 수준 |
| **문서 무결성** | 전자문서법 제4조 | AES-256-GCM 암호화 저장, DB 직접 수정 불가 | **○** | 암호화된 데이터는 키 없이 복호화·변조 불가 |
| **송수신 기록** | 전자문서법 제5조 | Supabase `tool4_audit_logs` 테이블에 IP, UA, 타임스탬프 기록 | **○** | 제출 시각, IP, User-Agent 기록으로 송수신 사실 증명 가능 |
| **보관/보존** | 전자문서법 제7조 | Supabase Cloud DB (AWS 서울/버지니아) | **○** | PostgreSQL 기반 영구 보관. 백업은 Supabase 자동 백업에 의존 |
| **열람 가능성** | 전자문서법 제7조 | `/result` 페이지에서 복호화·열람·인쇄 가능 | **○** | URL Fragment 내 암호키로만 접근. 키 분실 시 복호화 불가(설계 의도) |

### 1.3 전자서명 세부 분석

```
현재 구현된 서명 메커니즘:
┌─────────────────────────────────────────────────┐
│ 1) 보호자 성명 직접 입력 (consentGuardianName)  │
│ 2) "동의합니다" 확인 문구 타이핑               │
│ 3) 선택: 터치/마우스 자필 서명 (SignaturePad)   │
│ 4) guardianNameHash (SHA-256) 서버 저장         │
│ 5) IP + User-Agent + 타임스탬프 감사 로그       │
└─────────────────────────────────────────────────┘
```

| 서명 수준 | 설명 | 해당 여부 |
|---|---|:---:|
| **Level 1** (단순 전자서명) | 성명 입력 + 동의 버튼 | **해당** |
| **Level 2** (고급 전자서명) | 서명자 고유 식별 + 서명 데이터 연결 | **부분 해당** (SHA-256 해시 + IP 기록) |
| **Level 3** (공인 전자서명) | 공인인증기관 발급 인증서 기반 | **미해당** |

> **평가**: 교육 현장에서 사용되는 종이 동의서(보호자 자필 서명란)와 **동등 수준 이상**의 전자서명을 제공합니다. 공인전자서명은 미적용이나, 특수교육법상 동의서에 공인전자서명을 요구하는 규정은 없으므로 **실무적으로 충분**합니다.

| 하 | 정보보호 책임자 연락처 | 학교 개인정보보호 담당자 연락처 표기 (일부 교육청 감사 시 확인 항목) |

## 3부. 기술 보안 아키텍처 요약

```
┌──────────────────────────────────────────────────────────────┐
│                    IEP-ON Zero-Knowledge 아키텍처            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  [교사]                                                      │
│    │ 학교·학생 정보 입력                                      │
│    ▼                                                         │
│  POST /api/iep → Supabase (teacher_data 평문 JSONB)          │
│    │ iepId 생성 → QR코드 포함 안내장 출력                     │
│    ▼                                                         │
│  [보호자] QR 스캔 → /form?iepId=xxx                          │
│    │ 의견서 + 동의서 작성                                     │
│    ▼                                                         │
│  브라우저에서:                                                │
│    ├ AES-256-GCM 키 생성 (Web Crypto API)                    │
│    ├ 의견서 JSON → 암호화 → opinionEncrypted + opinionIv     │
│    ├ 동의서 JSON → 암호화 → consentEncrypted + consentIv     │
│    ├ guardianName → SHA-256 해시                              │
│    └ 서명 이미지 → Supabase Storage 업로드                   │
│    ▼                                                         │
│  POST /api/iep/{id}/submit → Supabase (암호문만 저장)        │
│    ▼                                                         │
│  리다이렉트 → /result?iepId=xxx#key=<256비트 hex>            │
│                                    ↑                         │
│                          URL Fragment (서버 미전송)           │
│                                                              │
│  [결과 열람] 브라우저에서 #key로 복호화 → 3개 문서 렌더링     │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│  핵심: 서버(Supabase)는 암호화 키를 절대 보지 못함            │
│  → DB 유출 시에도 개인정보 평문 노출 불가 (Zero-Knowledge)    │
└──────────────────────────────────────────────────────────────┘
```

## 4부. 종합 결론 및 권고

### 전체 평가
본 시스템은 **특수교육 현장의 IEP 수립 과정에서 보호자 의견서·동의서를 전자적으로 수집하는 도구**로서, 다음과 같은 수준을 달성하였습니다:

| 항목 | 평가 |
|---|---|
| **전자계약 요건 부합** | 우수 (4.3/5.0) — 작성자 확인·무결성·감사 추적·보관 요건 충족 |
| **문서 내용 적합성** | 매우 적합 — 특수교육법 시행규칙 양식 이상의 포괄적 항목 구성 |
| **개인정보보호** | 탁월 — Zero-Knowledge 암호화로 서버 측 평문 노출 원천 차단 |
| **실무 활용 가능성** | 높음 — QR코드 배포, 스마트폰 작성, A4 인쇄 출력 모두 지원 |

|:---:|---|:---:|:---:|
